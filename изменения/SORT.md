
## ДОДАНО (в обидва файли — однаковий код)
Локальний компаратор `compareTrips` + хелпер `_ts`:
- active / waiting → за зростанням `start_at` (найближчі/ранні спершу);
- completed        → за спаданням `end_at` (нещодавно завершені спершу), fallback `start_at`;
- спершу групування за статусом (active → completed → waiting) — статуси НЕ змішуються;
- подорожі без дати — в кінці своєї групи.

## ЗМІНЕНО

### AllTripsPage.jsx
- БУЛО: `.sort((a,b) => (statusWeight[a.status]||99) - (statusWeight[b.status]||99))`
  (тільки за статусом).
- СТАЛО: `.sort(compareTrips)` (статус + дата всередині групи).

### StatisticsPage.jsx
- БУЛО: сортування таблиці «Всі подорожі» за статусом + назва за алфавітом.
- СТАЛО: `.sort(compareTrips)` — та сама логіка, що на «Усі подорожі».
- ДОДАНО підвантаження реальних подорожей: `getTrips()` (бо `/api/statistic/trips`
  НЕ віддає дат). Рядок таблиці зіставляється з реальною подорожжю за назвою →
  беруться справжні `status`, `start_at`, `end_at`, далі сортування `compareTrips`.

## По якому полю сортуємо
- active / waiting → `start_at`
- completed        → `end_at` (fallback `start_at`)

## Endpoints
- `GET /api/content/get/trips` (через `getTrips`) — статуси та дати.
- `GET /api/statistic/trips` (через `stats.summary`) — дані таблиці статистики.

## Перевірено
- Логіка сортування однакова на «Усі подорожі» і в статистиці.
- Дату в інтерфейсі додатково НЕ показуємо — змінено лише порядок.
- Подорожі без дати → в кінець групи (без помилок).
- Часовий пояс: порівняння через `getTime()` (UTC), без зсувів.
- Збірка (vite build) проходить.

## Нюанс
- Компаратор тепер продубльовано у 2 файлах (бо без спільного util).
  Якщо змінювати правило — правити в обох місцях.
- Зіставлення статистики з реальними подорожами — за назвою. При однакових назвах
  дати візьмуться від першого збігу.
